## Architecture

Scout is a modular monorepo. Every layer has one job.

| Layer | Technology | Responsibility |
|:---|:---|:---|
| **Frontend UI** | Next.js (App Router), Tailwind CSS | Research-oriented chat interface with live job tracking, source drawers, and runtime trace logs |
| **Central API** | Fastify, TypeScript, Prisma ORM | Orchestrates project scopes, dispatches background jobs, manages semantic document chunks |
| **Task Queue** | Redis, BullMQ | Decouples long-running research from request lifecycles вҖ” deterministic status polling |
| **Background Worker** | Node.js | Drives job state transitions: `QUEUED вҶ’ RUNNING вҶ’ COMPLETED / FAILED` |
| **Scout Runtime** | Deno + Pyodide (WASM) | Translates LLM reasoning into live Python вҖ” calls `search_kb()` and `web_research()` recursively |
| **Model Layer** | NVIDIA Triton / OpenRouter | Specialized endpoints: reasoning (`glm4.7`), code gen (`qwen3-coder-480b`), embeddings (`nv-embedqa-e5-v5`) |
| **Web Ingestion** | Scraping | Deep scraping of public documentation into clean, structured Markdown |
| **Vector Storage** | Qdrant | High-dimensional semantic search, isolated per `projectId` |
| **Relational Storage** | Supabase Postgres | Jobs, Documents, Chunks, Reports, and granular Agent Execution traces |

## Getting Started

### Prerequisites

- [Docker](https://www.docker.com/products/docker-desktop/) вүҘ 20.10.0  
- [Docker Compose](https://docs.docker.com/compose/) вүҘ v2.0.0

That's it. Everything else runs inside containers.

---

### Option 1 вҖ” One Command

```bash
chmod +x ./run.sh && ./run.sh
```

Cleans, rebuilds, and spins up the full stack in one shot.

---

### Option 2 вҖ” Docker Compose

```bash
# Build all service images
docker-compose build

# Start everything
docker-compose up

# Or in detached mode
docker-compose up -d
```

Once containers are live, services auto-interconnect, migrations run, and the system is fully operational.

## End-to-End Test

To trigger the full pipeline вҖ” web crawl, ingestion, semantic search, Python execution, and synthesis вҖ” run this query in the UI:

> *"Compare Meta Marketing API, Google Ads API, and TikTok Ads API for building an ads automation platform. What data can I access, what can I automate, what are the required permissions, rate-limit risks, and the best MVP implementation plan?"*

**What happens:**

1. **Intent Analysis** вҖ” Scout detects multi-platform comparison, flags need for external documentation
2. **KB Fallback** вҖ” Qdrant returns empty; Firecrawl dispatched to crawl official API docs
3. **Ingestion** вҖ” Tables parsed, embedded, split into chunks, indexed into Qdrant
4. **Execution Loop** вҖ” Pyodide runs comparative evaluation scripts dynamically
5. **Synthesis** вҖ” A clean comparison matrix surfaces with direct source links

## Roadmap

- [ ] **Distraction-free UI** вҖ” Typography-first redesign, collapsible source drawers, and hidden trace layouts by default.
- [ ] **SSE Token Streaming** вҖ” Live token-by-token generation featuring real-time execution step metrics.
- [ ] **Domain Prioritization** вҖ” Smart source ranking that elevates official developer subdomains over secondary blogs.
- [ ] **Entity-Claim Knowledge Graphs** вҖ” Extracting explicit code entities to map relational knowledge across graph networks.
- [ ] **Multi-Agent Swarm Orchestration** вҖ” Transitioning to a hierarchical multi-agent layer where specialized sub-agents handle parallel research, code execution, and verification tasks.
- [ ] **Extensible Connector Matrix** вҖ” Native integrations for deep ingestion across private workspaces like **Notion**, Slack, GitHub, and Google Drive.
- [ ] **Multi-Tier Memory Architecture** вҖ” A persistent hybrid memory engine combining short-term runtime states (Pyodide), episodic trace logs, and long-term vector-backed recall.
